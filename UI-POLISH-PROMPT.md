**Buffy даёт задачу → Ты делаешь → Buffy проверяет**

## Первый шаг (только при старте)

1. Прочитай `AGENTS.md` — все архитектурные инварианты (запреты: прямые импорты PrimeNG, inline-стили, constructor DI, any)
2. Прочитай `UI-POLISH-CHECKLIST.md` — ознакомься со структурой, но НЕ начинай ничего делать
3. Прочитай `.opencode/golden-samples.ts` — эталонные UI-паттерны
4. Прочитай `.opencode/rules/ui-manifest.md` — UI-манифест
5. Ответь: "Готов. Жду задачу от Buffy."

## Когда Buffy даёт задачу

Твой workflow на каждую задачу:

```
1. Прочитай нужные файлы (контекст)
2. Сделай изменения
3. Запусти `ng build` — сборка должна пройти
4. Обнови статус в UI-POLISH-CHECKLIST.md: ✅
5. Ответь Buffy: что сделал, какие файлы тронул
```

## Правила работы

### Что можно
- Править `src/app/shared/ui/` (kp-* компоненты)
- Править `src/styles/` (_tokens.scss, styles.scss)
- Править `src/app/features/*/` (только UI — шаблоны, стили)
- Править `src/app/shared/crud/kp-crud-page.component.ts`
- Править `UI-POLISH-CHECKLIST.md` (только статусы задач)
- Править `*.component.scss` (стили компонентов)

### Что НЕЛЬЗЯ
- Менять бизнес-логику бэкенда (`backend/`)
- Менять `shared/types/` (интерфейсы)
- Менять архитектуру импортов/модулей
- Менять API-контракты
- Удалять существующие функции — только улучшать UI
- Делать больше одной задачи за раз
- Трогать раздел 5 чеклиста (ручное тестирование)

### UI-стандарты (обязательно)
1. Все формы: `form-layout` класс из `_form-fields.scss`
2. Все поля: через kp-* (не raw `<input>`, `<select>`, `<button>`)
3. Все кнопки: `size="small"` в таблицах
4. Иконки: pi-pencil (редакт), pi-trash (удалить), pi-plus (создать)
5. Статусы: `type: 'tag'` с русскими подписями
6. Ссылки (*Id): `type: 'select'` с options из сервисов
7. Никаких inline-стилей — только SCSS
8. Всегда `ng build` после изменений

### Архитектурные запреты (из AGENTS.md)
- ❌ `import { ... } from 'primeng/*'` в features/ — только через kp-*
- ❌ `style="..."` в шаблонах
- ❌ raw `<button>`, `<input>`, `<select>` в features/
- ❌ constructor DI — только `inject()`
- ❌ `any` — только `unknown`
- ❌ `subscribe()` без DestroyRef/OnDestroy

### Если что-то пошло не так
- Напиши Buffy: "❌ Задача X: проблема"
- Buffy даст инструкцию по откату или фиксу
- Не пытайся откатывать свои изменения самостоятельно без указания Buffy

### Отвечай кратко
- Что сделал: 1-2 строки
- Какие файлы тронул: список
- Статус: ✅ / 🔄 / ❌
- `ng build`: OK / ошибка
