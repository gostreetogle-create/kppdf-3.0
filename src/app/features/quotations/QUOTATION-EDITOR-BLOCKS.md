# QuotationEditor — панель блоков и порядок секций

> Маршрут: `/quotations/:id` · компонент: `quotation-editor.component.ts`  
> Дата работ: 2026-05-27  
> Статус: внедрено, `ng build` OK

Документ фиксирует UX-доработки боковой панели настройки блоков документа (КП) и связанные изменения в `kp-split-text-card`.

---

## Цель

Сделать настройку текстовых блоков на листе A4 удобной и без дублирования: широкая панель справа от листа, перетаскивание блоков мышью, защита от случайного сдвига, понятные границы кнопок.

---

## Что сделано

### 1. Боковая панель блока (`editor__block-controls`)

- Расположение: **серая зона справа от листа A4** (не печатается, скрывается в `@media print`).
- Ширина: **`--kp-block-rail-width: 280px`** (на `editor__canvas-wrapper`).
- Секции:
  - **Выравнивание** — слева / по центру / справа (для заголовка и всех колонок текстового блока).
  - **Цвет текста** — свотчи: авто, чёрный, синий, красный, зелёный.
  - **Жирный** и **рамка** (цикл: нет → пунктир → сплошная).
  - **Текстовый блок:** колонка `+`, редактирование (карандаш).
  - **Разделитель:** отступ.
  - **Удалить блок** (если блоков > 1).
- Кнопки оформлены как **плитки со светлой рамкой** (`outlined` + SCSS `::ng-deep` в панели).
- Подписи секций («Выравнивание», «Цвет текста») — статичный текст, **не** всплывающие подсказки.

### 2. Убрано дублирование

| Было | Стало |
|------|--------|
| Вертикальная панель у каждой колонки (выравнивание, edit, delete) | Убрана из `kp-split-text-card` |
| `pTooltip` на кнопках и зонах перетаскивания | Удалены все `pTooltip` в редакторе, `TooltipModule` отключён |
| Выравнивание заголовка в диалоге «Настройки текстового блока» | Убрано (есть на боковой панели) |
| Кнопки «вверх / вниз» на панели | Убраны (порядок — перетаскиванием) |
| Кнопка «Перетащить» (☰) | Убрана |

**Редактирование колонки:** двойной щелчок по тексту в карточке → диалог колонки (выравнивание одной колонки остаётся в диалоге для многоколоночных блоков).

### 3. Перетаскивание блоков (drag-and-drop)

- Реализация: **Angular CDK** через `appKpSortableList` / `appKpSortableItem` / `appKpSortableHandle` (`shared/ui/kp-sortable`).
- **Зона захвата:** область контента блока (текстовая карточка, заголовок, разделитель, заголовок таблицы «Товары»).
- Ось: только **вертикаль** (`appKpSortableLockAxis="'y'"`).
- Курсор `grab` / `grabbing` только когда порядок **не зафиксирован**.

### 4. Замок порядка блоков

- Кнопка в **верхней панели** редактора (слева от «Шаблоны»):
  - `pi-lock` + «Порядок зафиксирован» — перетаскивание **выключено**.
  - `pi-lock-open` + «Порядок открыт» — перетаскивание **включено**.
- **По умолчанию:** замок **открыт** (`blocksReorderLocked = false`).
- При закрытом замке: `cdkDropListDisabled` / `cdkDragDisabled`, guard в `onBlockDropped`, курсор без «руки», класс `editor__canvas--reorder-locked`.

### 5. Стили блока в данных

В `EditorBlock.settings` и `IDocumentBlock.settings` (`shared/types/documentTemplate.interface.ts`):

- `color?: string`
- `backgroundColor?: string`
- `borderStyle?: 'none' | 'dashed' | 'solid'`

Применение: заголовок (`blockVisualStyles`), карточка `kp-split-text-card` (рамка, цвет, фон).

### 6. Отказоустойчивость

- `isValidBlockIndex()` — проверка индекса перед мутациями блока.
- `onBlockDropped` — игнор при замке и невалидных индексах.
- `removeBlock` — не удалить последний блок.
- `editTextBlock` / `editTextCell` — только для `type === 'text'`.
- `setBlockAlign` для текста синхронизирует `align` во **всех колонках** и в `settings`.

## Поведение для пользователя (кратко)

1. Навести на блок → справа появляется панель настроек.
2. Менять выравнивание, цвет, жирный, рамку — на панели.
3. Двойной щелчок по тексту колонки — редактирование.
4. Чтобы **переставить** блок: убедиться, что в шапке замок **открыт** → зажать ЛКМ на тексте блока и тянуть.
5. Чтобы **не сдвинуть** случайно: закрыть замок в шапке.

## Технический долг / известное

- **PrimeNG в `QuotationEditor`** — допустимый tech debt (P0, см. `AGENTS.md`); панель блоков пока на `p-button`, не `kp-button`.
- **Кодировка файла:** `quotation-editor.component.ts` на Windows иногда сохраняется как **UTF-16** → ломает `ng build` (`Invalid character`). При ошибке сборки: пересохранить в UTF-8 или скрипт конвертации `utf16le` → `utf8`.
- **Персистенция замка** — только в сессии редактора, не в шаблоне/БД.
- **ProductPicker** и таблица товаров — отдельная интеграция (см. `docs/archive/analysis/AUDIT-PRODUCTPICKER.md`).
