# DESIGN Bridge

## Purpose

Сохранить ссылочную совместимость `DESIGN.md` после консолидации документации.

## Scope

- Этот файл больше не является самостоятельным canonical-документом.
- Подробная архитектурная карта и модель документации перенесены в `docs/ARCHITECTURE.md`.

## Source of Truth

- Архитектурная документация: `docs/ARCHITECTURE.md`
- Навигация по документации: `docs/INDEX.md`
- Инварианты и правила: `AGENTS.md`

## Main Content

`DESIGN.md` оставлен как bridge-док для обратной совместимости ссылок в старых чатах, планах и аудитах.  
Для актуальной работы по архитектуре и документационной системе используйте `docs/ARCHITECTURE.md`.
