# UI Audit Checklist

> Обязательный чеклист для `@ui-auditor`. Перед сдачей любой UI-работы — пройти по всем пунктам.

## 1. Кнопки (`p-button`)

| № | Проверка | Как проверить |
|---|----------|---------------|
| 1.1 | У всех кнопок указан `size` (`"small"` / `"large"`) | grep `p-button` в шаблонах — нет ли кнопок без `size` |
| 1.2 | severity указан явно (`primary` / `secondary` / `danger` / `success` / `warn`) | нет кнопок без `severity` (кроме первичных) |
| 1.3 | Кнопки одного размера на странице | визуально — или по классам в шаблоне |
| 1.4 | Icon-only кнопки имеют `severity` и `size="small"` | ✏️ 🗑️ — secondary/danger + small |
| 1.5 | У кнопок на тёмном фоне (сайдбар) отключён `transform: scale` | проверить `admin-layout.component.scss` |
| 1.6 | Все кнопки используют PrimeNG (`<p-button>`), никаких raw `<button>` | grep `<button` в шаблонах |
| 1.7 | Диалоговые кнопки (Отмена/Сохранить) помечены `size` | проверить `ng-template pTemplate="footer"` |

## 2. Формы и инпуты

| № | Проверка | Как проверить |
|---|----------|---------------|
| 2.1 | Все `<input>` имеют `pInputText` (не raw `<input>`) | grep `<input` — проверить атрибут |
| 2.2 | `pInputText`, `p-inputNumber`, `p-select` имеют `style="width:100%"` в диалогах | проверить диалоговые формы |
| 2.3 | Все контролы в форме выглядят одинаково (padding, font-size, border-radius) | сравнить SCSS-стили для `.p-inputtext`, `.p-inputnumber`, `.p-select` |
| 2.4 | У label указан `for` или структура `label + контрол` без gap-дыр | проверить `dialog-form__field` |
| 2.5 | Password-поля используют `<p-password>`, а не `<input type="password">` | grep `type="password"` |

## 3. Таблицы (`p-table`)

| № | Проверка | Как проверить |
|---|----------|---------------|
| 3.1 | Таблица использует `p-table`, не raw `<table>` | grep `<table` |
| 3.2 | Задан `styleClass` для таблицы (напр. `p-datatable-striped`) | проверить атрибут `styleClass` |
| 3.3 | Есть пагинация (`[paginator]="true"`) | проверить `p-table` атрибуты |
| 3.4 | Ячейки таблицы имеют consistent padding | проверить SCSS `.p-datatable-thead > tr > th` и `.p-datatable-tbody > tr > td` |
| 3.5 | Нет пустых/crashed ячеек (emptymessage есть) | проверить `ng-template pTemplate="emptymessage"` |
| 3.6 | Статусы используют `<p-tag>` | проверить `*ngSwitchCase="'tag'"` |

## 4. Диалоги (`p-dialog`)

| № | Проверка | Как проверить |
|---|----------|---------------|
| 4.1 | Диалог модальный (`[modal]="true"`) | проверить атрибут `modal` |
| 4.2 | Заголовок диалога передан (`[header]="..."`) | проверить атрибут `header` |
| 4.3 | Есть footer с кнопками действий | проверить `ng-template pTemplate="footer"` |
| 4.4 | Форма в диалоге однородна (все контролы одной ширины) | проверить inline-стили `style="width:100%"` |
| 4.5 | gap между полями формы одинаковый | проверить SCSS `.dialog-form` → `gap` |

## 5. Карточки (`p-card`)

| № | Проверка | Как проверить |
|---|----------|---------------|
| 5.1 | Карточка имеет header/subheader где нужно | проверить атрибуты `header` / `subheader` |
| 5.2 | Контент карточки использует `ng-template pTemplate="content"` | проверить шаблон |
| 5.3 | На dashboard карточки кликабельны и имеют hover-эффект | проверить `.dashboard__card:hover` |

## 6. Глобальные стили (`styles.scss`)

| № | Проверка | Как проверить |
|---|----------|---------------|
| 6.1 | Нет inline-стилей `style:` в компонентах | grep `style:` в SCSS |
| 6.2 | PrimeNG-токены в `app.config.ts` не конфликтуют с SCSS | проверить дублирующиеся `padding` / `border-radius` |
| 6.3 | Все PrimeNG-компоненты импортированы (не забыт ни один модуль) | grep `import.*from 'primeng/` в ts — сверить с используемыми |
| 6.4 | CSS-переменные (`--color-*`, `--radius-*`) используются консистентно | grep `#...` в SCSS — не захардкожены ли цвета |
| 6.5 | Favicon не 404 | проверить `<link rel="icon">` в `index.html` |

## 7. Специфичные для страниц

| № | Страница | Проверка |
|---|----------|----------|
| 7.1 | Login | Кнопка «Войти» full-width, размер small |
| 7.2 | Login | Password-поле с `[feedback]="false"` |
| 7.3 | 404 | Кнопка «На главную» размер small |
| 7.4 | Dashboard | Карточки используют `routerLink` |
| 7.5 | Dashboard | Есть skeleton-загрузка |
| 7.6 | Directories | Табы (активный/неактивный) корректно размечены |
| 7.7 | Directories | Action-колонка (карандаш/корзина) не разваливается |
| 7.8 | Admin Layout | Logout-кнопка корректно стилизована под тёмный фон |
| 7.9 | Admin Layout | Sidebar пункты имеют `routerLinkActive` |

## 8. Сборка

| № | Проверка |
|---|----------|
| 8.1 | `ng build` проходит без ошибок |
| 8.2 | Нет превышения бюджетов CSS (проверить warnings) |
| 8.3 | `ng lint` проходит без ошибок (если настроен) |

---

## Формат отчёта

После проверки `@ui-auditor` возвращает:

```
## UI Audit Report

### Проверено: [страница1, страница2, ...]

| № | Статус | Что проверил |
|---|--------|--------------|
| 1.1 | ✅ | Все кнопки имеют size |
| 1.2 | ⚠️ | Найден p-button без severity, исправлено |
| ... | ... | ... |

### Итого: ✅ N ок / ⚠️ M исправлено / ❌ 0 пропущено
```
