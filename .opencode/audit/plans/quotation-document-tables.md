> **Статус:** черновик для будущей реализации.  
> **Сейчас (2026-05):** в меню «+» только выпадающий список с одним пунктом **Товары**.

## Проблема

Пользователь может удалить блок «Товары» (таблица позиций КП). Нужно уметь вернуть таблицу и в перспективе — выбирать тип таблицы из настроек системы.

---

- [x] Меню добавления блока: секция **Таблица** + `p-select` + кнопка «Добавить таблицу».
- [x] Единственный тип: `products` → заголовок «Товары», рендер существующей таблицы позиций (`items()`).

## Фаза 1 — типы таблиц в shared/types + BE

1. `IDocumentBlock.tableKind?: string` (enum или slug из конфига).
2. Mongoose: поле `tableKind` в `documentBlockSchema` при `type: 'table'`.
3. Seed / дефолтный шаблон КП: `tableKind: 'products'`.

---

## Фаза 2 — настройки приложения (админка)

Раздел **Администрирование → Таблицы документов** (или подраздел «Шаблоны документов»):

| Поле | Описание |
|------|----------|
| `slug` | `products`, `services`, … |
| `label` | Подпись в выпадающем списке редактора |
| `docTypes` | `quotation`, `invoice`, … |
| `columns` | Конфиг колонок (ключ, заголовок, ширина, тип) |
| `dataSource` | `quotation.items`, API, статика |
| `permPrefix` | RBAC |

API: `GET /document-table-types?docType=quotation` → список для выпадающего списка в редакторе.

## Фаза 3 — QuotationEditor

1. Загрузка опций таблиц из API (fallback на `[{ kind: 'products', label: 'Товары' }]`).

3. Запрет второй таблицы «Товары» на документ — опционально, флаг в конфиге типа.
