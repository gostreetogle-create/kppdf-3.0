### 1.3 Основные бизнес-процессы (как это работает сейчас)

```
1. ПОЛУЧЕНИЕ ЗАПРОСА
   ↓
2. РЕГИСТРАЦИЯ В ЖУРНАЛЕ (mail_data)
   ↓
3. ПОДБОР ТОВАРОВ ИЗ ПРАЙСА (products)
   ↓
4. РАСЧЁТ СТОИМОСТИ (цена × кол-во + доставка + установка)
   ↓
5. ФОРМИРОВАНИЕ КП (коммерческого предложения)
   ↓
6. ОТПРАВКА КЛИЕНТУ
   ↓
7. ПРИ ПОБЕДЕ → ДОГОВОР → ЗАКАЗ → ПРОИЗВОДСТВО → ОТГРУЗКА
```

**Болевые точки текущего процесса:**
- ❌ Всё в Google Sheets — нет связности
- ❌ Ручной перенос данных между таблицами
- ❌ Нет автоматической генерации КП (редактор уже есть, но не интегрирован с тендерами)
- ❌ Нет истории взаимодействий с клиентами
- ❌ Нет контроля себестоимости

## 2. Архитектура KPPDF 3.0

### 2.1 Backend (Express + MongoDB)

**CRUD-фабрика** — унифицированный `createCrudRouter()` с:
- Пагинацией (?page, ?limit)
- Поиском (?search — regex)
- Сортировкой (?sort, ?order)
- Фильтром isActive (?all=true)
- Хуками beforeCreate / beforeUpdate

**Модули backend:**
