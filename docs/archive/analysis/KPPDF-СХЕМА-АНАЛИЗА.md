# Схема анализа папки «Для изучения»

## Метод

Анализирую **последовательно, папка за папкой**. Для каждого файла вытаскиваю:

1. **Сущность** (что это за объект: продукт, контрагент, заказ, и т.д.)
2. **Поля** (какие колонки/атрибуты есть у сущности)
3. **Связи** (с чем эта сущность связана: продукт ↔ заказ, контрагент ↔ КП, и т.д.)
4. **Паттерн** (повторяющийся приём: "номер формируется автоматически", "статус = enum", "сумма = кол-во × цена")

После анализа всех папок — свожу закономерности в общий паттерн-файл.

---

## Оглавление результатов

| # | Файл-вывод | Что содержит |
|---|------------|-------------|
| 1 | `АНАЛИЗ-01-КП.md` | Коммерческие предложения: каталог, настройки компаний, журнал запросов |
| 2 | `АНАЛИЗ-02-ПАСПОРТА.md` | Паспорта изделий: карточки продуктов |
| 3 | `АНАЛИЗ-03-ТЕХОПИСАНИЯ.md` | Технические описания изделий |
| 4 | `АНАЛИЗ-04-ПАТТЕРНЫ.md` | Общие паттерны и закономерности |

---

## Процесс анализа одного файла

```
┌─────────────────────────────────────────────────────┐
│ 1. Читаю HTML (Google Sheets экспорт)                │
│ 2. Определяю: какой бизнес-процесс описывает?        │
│ 3. Вытаскиваю сущности + поля                        │
│ 4. Определяю связи с другими таблицами               │
│ 5. Фиксирую повторяющиеся паттерны                   │
│ 6. Записываю в соответствующий файл-вывод            │
└─────────────────────────────────────────────────────┘
```
