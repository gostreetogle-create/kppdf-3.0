# YouGile — правила оформления задач KPPDF 3.0

> Единый стандарт доски «Статус реализации»: названия, иерархия, видимость на колонке, описания.

---

## Реестр PLM-xxx

**ID карточки = то, что в ссылке YouGile** (`#PLM-126`). Отдельный номер не нужен.

| Файл | Назначение |
|------|------------|
| `config/yougile-task-registry.yaml` | PLM-xxx → UUID, иерархия `subtasks` |
| `tools/yougile-registry.js` | show / resolve / repair-links |

```bash
node tools/yougile-registry.js show PLM-126
node tools/yougile-registry.js resolve PLM-126
node tools/yougile-registry.js repair-links
```

**В Cursor:** вставьте ссылку или напишите `Задача PLM-126 — …`

**Новая карточка:** создайте в YouGile → добавьте `PLM-xxx` в реестр → `repair-links`.

Названия карточек и колонок меняйте как угодно — скрипты смотрят на PLM и UUID.

## Доска и колонка

| Параметр | Значение |
|----------|----------|
| Доска | **Статус реализации** (`16d98239-919e-43f7-9bf4-a3fbadbdd580`) |
| Колонка модулей | **📋 План развития** (`59187569-feed-4a67-94ad-e20e7a098be7`) |
| Другие колонки | 🔄 В работе, ⬜ Не начато, ✅ Завершено — для операционных задач, не для дерева модулей |

На колонке «План развития» должны быть видны **только категории верхнего уровня** (`01 CRM` … `08 Документы`, `api`). Всё остальное — внутри карточек.

---

## Три уровня карточек

```
NN Модуль: название              ← категория (на доске)
└─ [EPIC #TAG] Название EPIC     ← группа фич (внутри категории)
    ├─ N.1 Задача верхнего уровня
    │   └─ N.1.1 Подзадача
    └─ N.2 Следующая задача
```

| Уровень | Формат названия | Пример | `columnId` |
|---------|-----------------|--------|------------|
| **Категория** | `NN DOMAIN: тема` | `01 CRM: Клиенты и КП` | ✅ колонка «План развития» |
| **EPIC** | `[EPIC #TAG] описание` | `[EPIC #CRM] Управление клиентами…` | ❌ не задавать |
| **Задача** | `N.x Краткое имя` | `1.4 UI: форма КП с таблицей товаров` | ❌ не задавать |
| **Подзадача** | `N.x.y Деталь` | `8.1.1 Шаблоны блоков документа` | ❌ не задавать |

**Тег EPIC:** `#CRM`, `#DOCS`, `#PLM`, `#ERP` — короткий идентификатор домена в квадратных скобках.

**Нумерация:** префикс совпадает с номером модуля — `1.x` в CRM, `2.x` в «02 CRM: Заказы», `8.x` в «08 Документы».

## Видимость на доске (`columnId`)

YouGile показывает в колонке **каждую задачу с `columnId` этой колонки**, даже если она уже вложена в родителя через `subtasks`.

| Правило | Детали |
|---------|--------|
| На доске | только **категории** (`01`–`08`, `api`) |
| Внутри карточки | EPIC, задачи `N.x`, подзадачи `N.x.y` — **без `columnId`** |
| POST вложенной | `{ "title": "…", "description": "…" }` — **без** `columnId` |
| POST категории | `{ "title": "…", "description": "…", "columnId": "<план>" }` |
| Сброс `columnId` | через PUT **нельзя** — пересоздать задачу или `fix-documents-board-visibility.js` |

**Симптом ошибки:** на доске торчат `8.1.1`, `8.2`, `[EPIC #DOCS]` отдельными карточками вместо одной «08 Документы».

## Иерархия (`subtasks`)

Связь родитель → дети **только** через массив `subtasks` у родителя:

```javascript
// 1. Создать вложенную задачу БЕЗ columnId
const child = await api.post("/tasks", { title: "8.1.1 …", description: "…" });

// 2. Добавить id в subtasks родителя
await api.put(`/tasks/${parentId}`, {
  subtasks: [...(parent.subtasks || []), child.id],
});
```

| Не работает | Работает |
|-------------|----------|
| `parentTaskId` в POST / PUT (400) | `subtasks: ["uuid", …]` у родителя |
| Дублировать ребёнка у двух родителей | Один родитель — один массив `subtasks` |

## Шаблон описания карточки

Для задач, привязанных к коду приложения:

```markdown
**Категория:** 08 Документы
**EPIC:** [EPIC #DOCS] Редакторы и печатные формы

Кратко: что делает эта задача (1–2 предложения).

**Маршрут:** `/quotations/:id`
**Dev:** http://localhost:4200/quotations/6a16902ab80b864c10674123

**Код:** `src/app/features/quotations/quotation-editor.component.ts`
**UX:** `src/app/features/quotations/QUOTATION-EDITOR-BLOCKS.md`

Связанные задачи: **1.4 UI** (список КП), **1.3 API** (CRUD).
```

| Поле | Когда заполнять |
|------|-----------------|
| Категория / EPIC | всегда для задач `N.x` |
| Маршрут, Dev | UI-разделы Angular |
| Код, UX | есть файлы в репозитории |
| Связанные задачи | перекрёстные ссылки между модулями |

Описание — Markdown. Прогресс EPIC обновляет `yougile-sync-server` автоматически.

## Чеклист: новая задача через API

1. Определить **категорию** (`01`–`08`) и номер **`N.x`** / **`N.x.y`**.
2. Найти **родителя** (EPIC или задачу `N.x`).
3. `POST /tasks` — **без `columnId`** для EPIC и ниже.
4. `PUT /tasks/:parentId` — дописать `id` в `subtasks`.
5. Заполнить **описание** по шаблону выше.
6. Проверить доску: лишних карточек на «Плане» быть не должно.

---

## Чеклист: вручную в UI YouGile

1. Открыть **категорию** (например `08 Документы`).
2. Добавить **стикер** (подзадачу) — не новую карточку в колонке.
3. Именовать по схеме `N.x` / `N.x.y`.
4. Не перетаскивать подзадачи в колонку «План разvития» как самостоятельные карточки.
5. В описании — маршрут, dev-ссылка, путь в репозитории.

## Типичные ошибки

| Ошибка | Как выглядит | Исправление |
|--------|--------------|-------------|
| `columnId` у подзадачи | 8.x на доске рядом с `08 Документы` | `node tools/fix-documents-board-visibility.js` |
| CRM-номер в другом модуле | `1.4.1` под `08 Документы` | переименовать в `8.1`, перенести в EPIC #DOCS |
| Два родителя в `subtasks` | одна карточка в CRM и в 08 | оставить одного родителя, у второго убрать id из `subtasks` |
| `parentTaskId` в API | 400 Bad Request | только `subtasks` у родителя |
